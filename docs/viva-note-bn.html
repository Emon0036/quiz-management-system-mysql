1 minute introduction

“আমার project টা একটা role-based online quiz management system। এখানে admin teacher manage করে, teacher quiz create/publish করে, আর student enroll করে exam দেয়, result দেখে, leaderboard আর progress track করে। আমি শুধু MCQ না, true/false, short answer, আর coding test / coding submission flow-ও রেখেছি। পুরো system টা Express + EJS + MySQL দিয়ে build করা হয়েছে।”

Why MVC style?

  • Code clean থাকে
  • Logic আর UI আলাদা থাকে
  • Maintenance easy হয়
  • New feature add করা সহজ
  • Controller reuse করা যায়
Viva line: “আমি MVC-like separation follow করেছি যাতে route, business logic, database schema আর view একসাথে জট না পাকায়।”

Core Logic

How exam evaluation works

Auto-graded flow

  • MCQ and true/false immediately compare করা যায়
  • Score এবং percentage সাথে সাথে calculate হয়
  • Pass/fail check করা হয়
  • Result save হয়
  • Leaderboard update হয়

Manual review flow

  • Short answer or coding answer initially pending-review থাকে
  • Teacher review page থেকে marks আর comments দেয়
  • তারপর final score update হয়
  • Then progress and ranking finalize করা হয়

Progress + points system

“আমি শুধু raw score store করি নাই। Student-এর overall growth দেখানোর জন্য average score, total points, streak, passed/failed count, category-wise progress এবং global leaderboard maintain করেছি। Hard quiz হলে points multiplier বেশি হয়, তাই difficulty অনুযায়ী reward system আছে।”

Possible viva questions with short answers

1. কেন MySQL use করেছ?
কারণ quiz, enrollment, attempt, result এগুলোর মধ্যে relational connection আছে। Sequelize দিয়ে schema, relation-style query, আর table sync manage করা সহজ হয়েছে।
2. কেন EJS use করেছ, React না?
এই project-এর জন্য server-rendered flow enough ছিল। EJS দিয়ে দ্রুত page rendering, form handling, and MVC integration সহজ হয়েছে।
3. Auto grading আর manual review দুটো কেন লাগল?
সব answer machine evaluate করতে পারে না। MCQ/true-false auto-check হয়, কিন্তু short answer আর coding submission teacher judgement দরকার।
4. Teacher-wise filter কেন important?
একই platform-এ অনেক teacher exam দিলে student confuse হতে পারে। Teacher filter দিলে student intended teacher-এর exam সহজে খুঁজে পায়।
5. Security কীভাবে handle করেছ?
Password hash with bcrypt, Passport auth, role middleware, blocked-account enforcement, MySQL session store, and route-level protection use করেছি।
6. Project-এর unique point কী?
শুধু quiz platform না, এখানে role approval, manual review, coding module, leaderboard, progress, multi-tab login handling, and teacher/category based browsing একসাথে আছে।
